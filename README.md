# 🔌 Communication Protocols in Embedded Systems

This project demonstrates the implementation of common **communication protocols used in embedded systems and microcontrollers**.

Communication protocols allow different hardware components to exchange data efficiently in embedded devices.

This repository includes working examples of:

* **SPI (Serial Peripheral Interface)**
* **I2C (Inter-Integrated Circuit)**
* **UART (Universal Asynchronous Receiver Transmitter)**

These protocols are widely used in development boards such as **Arduino, ESP32, and STM32**.

---

# 📌 Project Overview

Embedded systems rely on communication protocols to transfer data between:

* Microcontrollers
* Sensors
* Displays
* Memory modules
* Other peripheral devices

This project demonstrates how these protocols work through simple implementation examples.

---

# 🚀 Protocols Implemented

### 🔹 SPI (Serial Peripheral Interface)

SPI is a **high-speed synchronous communication protocol** used for short-distance communication.

Features:

* Master-Slave communication
* High data transfer speed
* Full duplex communication

Common Uses:

* SD cards
* LCD displays
* Sensors
* Flash memory

---

### 🔹 I2C (Inter-Integrated Circuit)

I2C is a **two-wire communication protocol** widely used for connecting multiple peripherals.

Features:

* Only **two wires required (SDA & SCL)**
* Supports multiple devices
* Master-slave architecture

Common Uses:

* Temperature sensors
* RTC modules
* EEPROM memory
* OLED displays

---

### 🔹 UART (Universal Asynchronous Receiver Transmitter)

UART is an **asynchronous serial communication protocol** used for device-to-device communication.

Features:

* No clock signal required
* Simple implementation
* Widely used for debugging and data transfer

Common Uses:

* Serial communication with PC
* GPS modules
* Bluetooth modules
* Debugging microcontrollers
